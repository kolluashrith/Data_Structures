The optimizations in ArrayBag and LinkedListBag aim to reduce the time it takes to access frequently accessed elements. They do this by moving an element to the front (LinkedListBag) or closer to the front (ArrayBag) every time it is searched for. This means that subsequent searches for the same element will find it quicker since it is moved closer to the front of the list being traversed. An example situation in which this would be helpful is an application where the user is searching through a list of contacts. A person's contacts list is typically very large containing the numbers of many people. However, when people use their contacts, they typically search for the same set of people frequently while rarely searching for others (i.e. close family and friends versus the guy from high school they worked with on a group project many years ago). Thus in this case, a move-to-front heuristic would be extremely helpful because it would greatly reduce the average search time since most of the people who will be searched for will be near the front of the list.
